Original question: Of the twitter accounts tweeting the hashtag #StoptheSteal2020, how many of those accounts are bots?¶

[For this project, I will select a currently trending and relevant hashtag on twitter, and then train a classifier to predict whether or not the accounts tweeting that hashtag are real people, or bots. Politics is a topic on social media that is targeted by malicious actors that use bots to manuipulate how much a particular topic trends. I decided to pick the Twitter hashtag #StopTheSteal2020. In my opinion, malicious actors that manipulate social media to facilitate the spread of misinformation are a serious threat to human prosperity. My hope is that I will be able to use the tools I've learned in this class to investigate the role of twitter bots spreading misinformation about the 2020 Election. This project is original because I will be testing my classifier and forming conclusions on data that I extracted myself using the tweepy api]

Data (writing)¶

I will train my classifier on a dataset obtained on kaggle here: https://www.kaggle.com/charvijain27/detecting-twitter-bot-data This dataset contains detailed information about specific Twitter users. The features consist of various types of info about specific user accounts, and the labels indicate whether or not the account is a bot.

Once I have my model trained on the above data, I will run it on a custom dataset that I gathered from twitter using the tweepy api. The custom dataset will have the same feature set as the training dataset, but the actual data will be from users tweeting the hashtag "StopTheSteal2020". There will be no way of knowing for sure if the model is correctly predicting if the users tweeting "StopTheSteal2020" are bots, but it will be interesting to see the percentage of users the model predicts to be bots. Assuming that I build a model that is reasonably accurate, I should be able to get a ballpark estimate of which percentage of accounts tweeting StopTheSteal2020 are acutally real people vs bots.

Here are some examples of features I will be working with: followers_count, friends_count, favorites_count, verified, statuses_count, default_profile, default_profile_image

class MyStreamListener(tweepy.StreamListener):
   
    def __init__(self):
        self.output_file = open('tweets.json', 'w',encoding = 'utf-8')
# add a newline character if there isn't one already       
    def on_data(self, data):
        if not data.endswith('\n'):    
            data += '\n'
#write the data
        self.output_file.write(data)
#error handling        
    def on_error(self, status):
        print('ERROR', status)
        return False
            
myListener = MyStreamListener()

K-Nearest Neighbors¶

Knn could be a good fit for this data because it is possible that bots have similar characteristics as other bots and same for humans. These possible feature similarities could result in distinguishable clusters. If this is the case, knn would be an effective classifier due to clustering of users. Knn only has 1 hyperparameter, which is the value for k, or the number of neighbors. A reason not to chose knn for this dataset would be that there are many features, and knn can struggle with high dimensionality.

Results¶

With a 83% testing accuracy, it seemed like knn would also be a viable opiton. However, the classification report and confusion matrix revealed problems. The opposite is true for class 1. The lower f1 scores reflect the inconsistencies with precision and recall.

Random forest gave the highest test accuracy of all 3 classifiers. Random forest also had the highest f1 scores of the 3 models. It appears that random forest is the best classifier for this data/feature set.
